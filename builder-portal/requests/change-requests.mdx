The Change Requests section shows all client requests for project modifications across your organization. It provides a formal funnel from client idea to evaluated change order, with full visibility for both project teams and management.

## Request list

View all change requests with:

| Column | Description |
|--------|-------------|
| **Title** | Request description |
| **Project** | Associated project |
| **Client** | Who submitted it |
| **Status** | Current status |
| **Priority** | Internal priority (for example, Low, Normal, High) |
| **Date** | When submitted |
| **Impact** | Estimated cost/schedule impact (once evaluated) |

## Request status

Change requests move through stages:

| Status | Description |
|--------|-------------|
| **Submitted** | Request received |
| **Under review** | Being evaluated |
| **Quoted** | Cost estimate provided |
| **Approved** | Client approved the change |
| **In progress** | Work underway |
| **Completed** | Change implemented |
| **Declined** | Request rejected |

## Viewing request details

Click on any request to see:

- Full description from the client.
- Attached files and images.
- Client comments and message history.
- Internal evaluation notes and status history.
- Cost estimate and proposed pricing.
- Schedule impact.

## Processing a change request

### 1. Review the request

- Read the description carefully
- Review attached photos or documents
- Ask clarifying questions if needed

### 2. Evaluate impact

- Assess cost implications
- Determine schedule impact
- Identify any constraints

### 3. Prepare response

- Create a cost estimate.
- Document scope changes in clear language.
- Note any timeline adjustments.

### 4. Communicate with client

- Send the estimate or response.
- Answer questions using the request’s localized chat.
- Obtain approval to proceed.

### 5. Execute

- Create a change order from the approved request.
- Schedule the work.
- Update the project schedule and budget as needed.

## Filtering requests

Filter by:

- **Project** — Specific project
- **Status** — Request stage
- **Priority** — Urgency level
- **Date** — Submission date range

<Tip>
Process change requests promptly. Delays can frustrate clients and impact project schedules.
</Tip>

## Converting to change orders

Approved change requests should be converted to formal change orders:

1. Review the approved request
2. Create change order from request
3. Include all scope and pricing
4. Send for client signature

<Note>
Maintain clear documentation of the change request to change order process for contract clarity.
</Note>
