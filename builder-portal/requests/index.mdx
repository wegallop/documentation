The Requests section provides a centralized view of all client requests across your projects. Track change requests and warranty claims efficiently.

Requests give clients a formal channel to raise scope changes and warranty issues, while giving your team a structured way to triage, assign, and resolve them. This avoids important requests getting lost in email, text, or informal chat.

## Overview

The Requests section helps you:

- Track all incoming requests
- Prioritize response and action
- Monitor request status
- Ensure timely resolution

## Request workflow

Both change and warranty requests follow a similar workflow:

1. **Submitted** — Client submits request
2. **Under review** — Team evaluates request
3. **Action taken** — Response or resolution provided
4. **Completed** — Request resolved

## Navigation

Use the tabs to switch between:

- **Change Requests** — Modifications to active projects
- **Warranty Requests** — Issues after completion

## Quick actions

From this section:

- View request details
- Assign team members
- Update status
- Communicate with clients

<Tip>
Respond to requests promptly, even if just to acknowledge receipt. Clients appreciate knowing their request is being addressed.
</Tip>

<Note>
New requests are submitted by clients through the Client Portal. This section provides the builder's view for management and response.
</Note>
