The Warranty Requests section shows all warranty claims from clients with active or completed projects. It gives you a structured way to triage issues, coordinate trades, and document resolutions over the life of your warranty obligations.

## Request list

View all warranty requests with:

| Column | Description |
|--------|-------------|
| **Issue** | Description of the problem |
| **Project** | Associated project |
| **Client** | Who submitted it |
| **Category** | Type of issue |
| **Status** | Current status |
| **Priority** | Internal urgency |
| **Date** | When submitted |

## Request status

Warranty requests progress through:

| Status | Description |
|--------|-------------|
| **Submitted** | Request received |
| **Under review** | Being evaluated |
| **Scheduled** | Repair scheduled |
| **In progress** | Work being done |
| **Completed** | Issue resolved |
| **Denied** | Not covered by warranty |

## Viewing request details

Click on any request to see:

- Full issue description.
- Photos of the problem.
- Location within the property.
- Category classification.
- Links to related selections or items (where applicable).
- Communication history and localized chat.
- Internal notes and resolution details.

## Processing warranty requests

### 1. Review the issue

- Examine photos and description
- Determine the nature of the problem
- Check warranty coverage

### 2. Evaluate coverage

- Review warranty terms
- Determine if issue is covered
- Document your assessment

### 3. Respond to client

- Acknowledge the request
- Communicate coverage status
- Provide timeline if covered

### 4. Schedule resolution

- Coordinate with trades
- Set repair appointment
- Notify client

### 5. Complete and close

- Perform repairs
- Document completion
- Close the request

## Warranty coverage

Common covered items:

- Structural defects
- Workmanship issues
- Material failures
- System malfunctions

Typically not covered:

- Normal wear and tear
- Client-caused damage
- Maintenance items
- Natural settling

## Filtering requests

Filter by:

- **Project** — Specific project
- **Category** — Type of issue
- **Status** — Request stage
- **Priority** — Urgency level

<Tip>
Respond to warranty requests within your stated timeframe, even if just to acknowledge receipt. Good warranty service builds reputation and referrals.
</Tip>

## Documentation

Maintain complete records of:

- Original issue photos
- Communication with client
- Repair work performed
- Completion documentation

<Note>
Warranty terms are defined in project settings. Ensure clients understand coverage before project completion.
</Note>
