Configuration settings define how your organization's data is structured and categorized. These shared lists drive projects, catalogue content, financials, and permissions across Gallop.

<Note>
Configuration settings are only accessible to administrators.
</Note>

## Organizational structure

Use these lists to mirror how your company is organized:

### Job titles

Define roles within your organization (for example, Project Manager, Superintendent, Designer, Estimator, Administrator). Job titles appear on user profiles and can be used for filtering and reporting.

### Departments

Group team members by department (for example, Operations, Design, Finance, Administration). Departments help you understand where work is happening in the business.

### Groups

Create **Groups** that you use to manage permissions (for example, Project Managers, Site Supervisors, Accounting). Groups can receive elevated permissions that cascade to all members. See the **Users** documentation for more on how groups affect access.

## Project configuration

These lists control how projects are classified and how schedules are structured:

### Project types

Define the types of projects you handle, such as:

- New construction
- Renovation
- Addition
- Commercial
- Residential

### Project stages

Set up your project lifecycle stages. A common pattern is:

- Prospect / Lead
- Active / Construction
- Warranty
- Post‑warranty
- Complete

You can adapt these to match your existing language. Stages appear in project lists and reports.

### Project phases

Define phases used on schedules and in client‑facing timelines, such as:

- Design
- Permits
- Foundation
- Framing
- MEP rough‑in
- Finishes
- Closeout

Phases are used when you build project schedules and briefings.

## Categorization

These lists power catalogue items, selections, and reporting:

### Categories

High‑level item categories that you use across the catalogue and decision tools, for example:

- Plumbing
- Electrical
- Flooring
- Finishes
- Appliances

### Rooms

Location categories for selections, tasks, and items, such as:

- Kitchen
- Primary bathroom
- Living room
- Exterior

### Tags

Create tags for flexible filtering and reporting:

- **Project tags** — For example, priority levels, client types, neighbourhoods, product lines.
- **File tags** — For example, document type, status, department.
- **Catalogue image tags** — For example, product type, style, brand.

## Financial configuration

These lists support project accounting and client financials:

### Cost codes

Your cost code structure for budgets, forecasts, costs, and cost reports. You can mirror an existing accounting or estimating framework, including divisions and sub‑codes.

### Payment methods

Accepted payment types your team uses on invoices and internal reports, such as:

- Check
- Bank transfer
- Credit card
- Online payment

### Billing accounts

Categories you use to group revenue or internal reporting lines, for example:

- By project type
- By service type
- By region or business unit

## Best practices

- **Be comprehensive** — Add the key lists you know you will need before creating projects.
- **Be consistent** — Use standard naming conventions so your team can filter and search reliably.
- **Plan ahead** — Align cost codes, billing accounts, and project stages with your accounting and reporting needs.
- **Keep it simple** — Start with a focused set of options and expand only when you have a clear need.
