Change order templates standardize how you document contract modifications. They are built on the same block system as proposals, but tuned for describing deltas to the contract rather than full scopes of work.

## Template list

View all change order templates:

| Column | Description |
|--------|-------------|
| **Name** | Template name |
| **Type** | Change type or use case |
| **Description** | Template purpose |

## What a change order template contains

A change order template typically includes:

- **Header information**
  - Auto‑generated change order number.
  - Project reference and original contract reference.
  - Date and, optionally, effective date.
- **Sections and items**
  - Standard **Section** and **Item** blocks for common change patterns (for example, upgrades, credits, substitutions).
  - Links to Catalogue items and collections you use frequently for changes.
- **Pricing and totals**
  - A **Price** block focused on net change and new contract total.
  - Clear breakdown of additions, credits, and net impact.
- **Terms and signatures**
  - Standard terms covering how the change affects price, schedule, and warranty.
  - Signature configuration for builder and client signatories.

## Creating and using templates

To create a new change order template:

1. Click **New template** in **Templates → Change orders**, or save from an existing change order.
2. Set the template name, description, and intended use (for example, upgrades, credits, unforeseen conditions).
3. Build or refine the canvas using the same blocks available in proposals.
4. Lock any language that should not be changed on a per‑change basis.
5. Save the template.

When creating a change order from a project:

1. Start from a relevant template.
2. Replace placeholder items with specific scope changes and pricing.
3. Adjust any unlocked text to describe the change precisely.
4. Review the impact on contract value and schedule.
5. Send for internal approval and then to the client for signature.

<Tip>
Include clear, standardized language in templates that explains how the change affects the original contract, timeline, and budget. This reduces back‑and‑forth and protects both you and your clients.
</Tip>
