Proposal templates define reusable starting points for your proposals. They capture canvas structure, block configuration, and standard language so your team can generate consistent, on‑brand documents quickly.

## Template list

View all proposal templates with:

| Column | Description |
|--------|-------------|
| **Name** | Template name |
| **Type** | Intended project type or use case |
| **Created** | When the template was created |
| **Last used** | Most recent proposal created from it |

## What a template contains

A proposal template can include:

- **Settings defaults**
  - Default title pattern and reference number format.
  - Default validity period and estimated duration.
  - Default preparer role or group.
- **Canvas layout**
  - Pre‑built **Section**, **Item**, **Contingency**, **Text**, **Image**, **Price**, **Terms**, and **Signature** blocks.
  - Standard ordering of sections (for example, Overview, Scope, Allowances, Price, Terms, Signatures).
- **Block configuration**
  - Inclusion/visibility/restriction settings for each block.
  - Tokens in text blocks for dynamic content (org, project, proposal, or item fields).
  - Links to Catalogue items, collections, text, images, and terms.

Templates do not lock you into a fixed structure; they provide a strong starting point that can be tuned per project.

## Creating a proposal template

There are two common ways to create templates:

1. **From scratch**
   1. Go to **Templates → Proposals** and click **New template**.
   2. Set the template name, description, and intended project type.
   3. Build out the canvas with the block types you want.
   4. Configure inclusion, visibility, and restriction on each block.
   5. Save the template.
2. **From an existing proposal**
   1. Open a proposal that reflects a pattern you want to reuse.
   2. Clean up client‑specific details while leaving structure and standard content intact.
   3. Use the **Save as template** action.
   4. Give the template a clear name and description.

In both cases, consider which content should remain locked (for example, terms) and which should be editable by project teams.

## Using templates when creating proposals

When you create a new proposal from a project:

1. Click **New proposal**.
2. Choose a template that matches the project type or scope.
3. Review the pre‑populated canvas:
   - Swap in catalogue items or collections where needed.
   - Adjust text blocks while leaving locked content intact.
   - Confirm terms and signature configuration.
4. Configure pricing and payment schedules.
5. Run the preflight checklist and send for review or directly to the client.

Each proposal becomes independent once created; subsequent edits to the template do not affect existing proposals.

## Maintaining templates

Keep templates aligned with how you actually sell work:

- **Review regularly** — Retire outdated templates and consolidate duplicates.
- **Update catalogue links** — Ensure items, collections, text, and terms referenced by templates are current.
- **Refine restrictions** — Lock down content that should not be edited on a per‑proposal basis.
- **Align with approvals** — Design templates to work well with your internal review workflows.

<Tip>
Invest time building a small number of excellent templates for your most common project types. This pays off every time a new proposal is created.
</Tip>
