Task templates define recurring tasks that appear on projects. Create templates for common internal workflows and for client‑facing tasks, so nothing is missed and everyone knows what to do next.

## Template list

View all task templates:

| Column | Description |
|--------|-------------|
| **Name** | Template name |
| **Tasks** | Number of tasks |
| **Category** | Task category (for example, Preconstruction, Closeout, Client tasks) |

## Creating templates

To create a new task template:

1. Click **New template**.
2. Enter template details:
   - Template name.
   - Description.
   - Category.
3. Add tasks:
   - Task titles.
   - Descriptions.
   - Default assignees or roles.
   - Relative due dates.
   - Whether each task is **client‑facing** (Client Task) or **internal only**.
4. Save the template.

## Template structure

### Template information

- **Name** — Template identifier.
- **Description** — When to use this template.
- **Category** — Grouping (for example, phase, department, or client).

### Individual tasks

Each task includes:

- **Title** — What needs to be done.
- **Description** — Details and instructions.
- **Assignee** — Who is responsible (role‑based or specific team member).
- **Due date** — When due (relative to project or phase dates).
- **Priority** — Importance level.
- **Attachments** — Reference documents or forms.
- **Client visibility** — Whether the task appears in the client’s **Decisions** view as a Client Task.

## Common task templates

### Preconstruction tasks

- Permit applications.
- Plan reviews.
- Material ordering.
- Subcontractor scheduling.

### Client tasks

Client‑facing tasks appear in the **Client Portal → Hub → Decisions** section. Common templates include:

- Selection deadlines.
- Document approvals.
- Site access coordination.
- Design sign‑offs.
- Move‑in or move‑out checklists.

### Phase completion tasks

- Inspections.
- Punch list items.
- Client walkthroughs.
- Documentation.

### Closeout tasks

- Final inspections.
- Certificate of occupancy.
- Warranty registration.
- Client training.

<Tip>
Create task templates for each project phase and for key client moments. Import them as the project progresses so both your team and your clients always know what comes next.
</Tip>

## Using templates

When adding tasks to a project:

1. Select a task template.
2. Import tasks.
3. Adjust dates for the project timeline.
4. Assign tasks to specific team members.
5. Activate tasks so they appear in the appropriate inboxes (builder or client).

## Relative dates

Task templates use relative dates:

- Project start + X days.
- Phase start + X days.
- Milestone − X days.
- Previous task + X days.

This allows templates to adapt to any project timeline.

<Note>
Relative dates are converted to actual dates when the template is applied to a project.
</Note>

## Best practices

- **Comprehensive lists** — Include all necessary tasks for a given phase or scenario.
- **Clear descriptions** — Explain what’s required and how to complete the task.
- **Realistic timelines** — Set achievable due dates based on your typical workflow.
- **Proper sequencing** — Order tasks logically and use relative dates to keep sequences in sync.
